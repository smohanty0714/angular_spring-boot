    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-basic-crud-app-with-angular-50-and-spring-boot-20" class="anchor" aria-hidden="true" href="#basic-crud-app-with-angular-50-and-spring-boot-20"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Basic CRUD App with Angular 5.0 and Spring Boot 2.0</h1>
<p>This example app shows how to build a basic CRUD app with Spring Boot 2.0, Spring Data, and Angular 5.0.</p>
<p>Please read <a href="https://developer.okta.com/blog/2017/12/04/basic-crud-angular-and-spring-boot" rel="nofollow">Build a Basic CRUD App with Angular 5.0 and Spring Boot 2.0</a> to see how this app was created.</p>
<p><strong>Prerequisites:</strong> <a href="http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html" rel="nofollow">Java 8</a> and <a href="https://nodejs.org/" rel="nofollow">Node.js</a>.</p>
<blockquote>
<p><a href="https://developer.okta.com/" rel="nofollow">Okta</a> has Authentication and User Management APIs that reduce development time with instant-on, scalable user infrastructure. Okta's intuitive API and expert support make it easy for developers to authenticate, manage and secure users and roles in any application.</p>

<h2><a id="user-content-getting-started" class="anchor" aria-hidden="true" href="#getting-started"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Getting Started</h2>
<p>To install this example application, run the following commands:</p>
<div class="highlight highlight-source-shell"><pre>git clone https://github.com/oktadeveloper/okta-spring-boot-2-angular-5-example.git
<span class="pl-c1">cd</span> okta-spring-boot-2-angular-5-example</pre></div>
<p>This will get a copy of the project installed locally. To install all of its dependencies and start each app, follow the instructions below.</p>
<p>To run the server, cd into the <code>server</code> folder and run:</p>
<div class="highlight highlight-source-shell"><pre>./mvnw spring-boot:run</pre></div>
<p>To run the client, cd into the <code>client</code> folder and run:</p>
<div class="highlight highlight-source-shell"><pre>npm install <span class="pl-k">&amp;&amp;</span> npm start</pre></div>
<h3><a id="user-content-create-an-oidc-app-in-okta" class="anchor" aria-hidden="true" href="#create-an-oidc-app-in-okta"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Create an OIDC App in Okta</h3>
<p>You will need to <a href="https://developer.okta.com/blog/2017/12/04/basic-crud-angular-and-spring-boot#create-an-oidc-app-in-okta" rel="nofollow">create an OIDC App in Okta</a> to get your values to perform authentication.</p>
<p>Log in to your Okta Developer account (or <a href="https://developer.okta.com/signup/" rel="nofollow">sign up</a> if you don’t have an account) and navigate to <strong>Applications</strong> &gt; <strong>Add Application</strong>. Click <strong>Single-Page App</strong>, click <strong>Next</strong>, and give the app a name you’ll remember. Change all instances of <code>localhost:8080</code> to <code>localhost:4200</code> and click <strong>Done</strong>.</p>
<h4><a id="user-content-server-configuration" class="anchor" aria-hidden="true" href="#server-configuration"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Server Configuration</h4>
<p>Set your domain and copy the <code>clientId</code> into <code>server/src/main/resources/application.yml</code>.</p>
<p><strong>NOTE:</strong> The value of <code>{yourOktaDomain}</code> should be something like <code>dev-123456.oktapreview</code>. Make sure you don't include <code>-admin</code> in the value!</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">security</span>:
    <span class="pl-ent">oauth2</span>:
        <span class="pl-ent">client</span>:
            <span class="pl-ent">access-token-uri</span>: <span class="pl-s">https://{yourOktaDomain}.com/oauth2/default/v1/token</span>
            <span class="pl-ent">user-authorization-uri</span>: <span class="pl-s">https://{yourOktaDomain}.com/oauth2/default/v1/authorize</span>
            <span class="pl-ent">client-id</span>: <span class="pl-s">{clientId}</span>
            <span class="pl-ent">scope</span>: <span class="pl-s">openid profile email</span>
        <span class="pl-ent">resource</span>:
            <span class="pl-ent">user-info-uri</span>: <span class="pl-s">https://{yourOktaDomain}.com/oauth2/default/v1/userinfo</span>
            <span class="pl-ent">token-info-uri</span>: <span class="pl-s">https://{yourOktaDomain}.com/oauth2/default/v1/introspect</span>
            <span class="pl-ent">prefer-token-info</span>: <span class="pl-c1">false</span></pre></div>

<p>For the client, set the <code>issuer</code> and copy the <code>clientId</code> into <code>client/src/app/app.module.ts</code>.</p>
<div class="highlight highlight-source-ts"><pre><span class="pl-k"><span class="pl-k">const</span></span> config <span class="pl-k">=</span> {
  issuer: <span class="pl-s"><span class="pl-pds">'</span>https://{yourOktaDomain}.com/oauth2/default<span class="pl-pds">'</span></span>,
  redirectUri: <span class="pl-c1">window</span>.<span class="pl-c1">location</span>.<span class="pl-smi">origin</span> <span class="pl-k">+</span> <span class="pl-s"><span class="pl-pds">'</span>/implicit/callback<span class="pl-pds">'</span></span>,
  clientId: <span class="pl-s"><span class="pl-pds">'</span>{clientId}<span class="pl-pds">'</span></span>
};</pre></div>
